Este repositório pode ser acessado online ou instalado localmente clonando do GitHub.
Ambas as opções permitem que você tenha uma base de conhecimento pronta para ser
consumida por agentes de IA.

## Uso Online

::steps
### Acessar a base de conhecimento online

A base de conhecimento está disponível online e pode ser acessada imediatamente
sem necessidade de instalação:

**URL**: <https://context-directory.quila.dev>

### Vantagens do uso online

- **Acesso imediato**: Não requer instalação ou configuração
- **Sempre atualizado**: Conteúdo sempre na versão mais recente
- **Disponível em qualquer lugar**: Acesso de qualquer dispositivo com internet
- **Ideal para agentes de IA**: Agentes podem acessar diretamente via URL para
  consumir o conteúdo

### Como agentes de IA podem usar

Agentes de IA podem acessar a documentação online diretamente através da URL.
O conteúdo está estruturado de forma a facilitar a leitura e processamento por
sistemas de IA, permitindo busca semântica e navegação contextual.

Para mais informações sobre como agentes de IA devem consumir este repositório,
consulte a documentação sobre [uso por agentes de IA](/primeiros-passos/para-agentes-ia).
::

## Instalação via GitHub

::steps
### Clonar o repositório

Clone o repositório do GitHub para sua máquina local:

```bash [Terminal]
git clone https://github.com/anilsonlopes/context-directory.git
```

Ou usando SSH:

```bash [Terminal]
git clone git@github.com:anilsonlopes/context-directory.git
```

### Navegar para o diretório

Entre no diretório do projeto:

```bash [Terminal]
cd context-directory
```

### Instalar dependências

Instale as dependências do projeto:

```bash [Terminal]
npm install
```

Ou usando outros gerenciadores de pacotes:

```bash [Terminal]
# Usando pnpm
pnpm install

# Usando yarn
yarn install

# Usando bun
bun install
```

### Iniciar servidor de desenvolvimento

Inicie o servidor de desenvolvimento para visualizar a documentação:

```bash [Terminal]
npm run dev
```

A documentação estará disponível em <http://localhost:3000>

### Personalizar o conteúdo

Edite os arquivos Markdown no diretório `content/` para personalizar a base
de conhecimento conforme suas necessidades.
::

## Próximos Passos

Após acessar online ou instalar localmente, você pode:

- **Personalizar o conteúdo**: Edite os arquivos Markdown em `content/` para

- **Adicionar novos documentos**: Siga as [convenções de manutenção](/primeiros-passos/manutencao-conteudo)
  para adicionar novos documentos
- **Configurar para agentes de IA**: Consulte a documentação sobre
  [uso por agentes de IA](/primeiros-passos/para-agentes-ia) para entender como
  otimizar o conteúdo

::tip
